\section{Saisie du Cycle}
 \subsection{Comment orienter une arête?}

   Des algorithmes de type Diskjtra, pour les simuler on a besoin de tracer un cycle dans le graphe, et d'orienter ce cyle. L'outil ViSiDia permet à l'utilisateur de tracer des arêtes orientés. \\
  Les arêtes sont par défaut non orientés. Il est touefois possible de tracer des arêtes orientés. \\
  Dans le menu Transformation, aller dans Change Edge Shape, une boite de dialogue apparait, à ce moment là, choisir le type d'arête qu 'on veut tracer et appuyer sur OK. \\
  Il est à signaler qu'on peut changer le type d'arête à tout moment durant le tracé du graphe. \\

 \subsection{Comment saisir un cycle}
  
 Si l'algorithme le nécessite, le mieux pour tracer un graphe avec cyle, et de commencer par tracer le cycle. i.e lier tout les sommets par avec des arêtes orientées d'abord, ensuite ajouter les arêtes non orientées s'il en faut.
 
 Il faut noter que derrière l'affichage, 
 Ceci dit, rien n'oblige à suivre cette règle.
 \subsection{Remarque importante}

  Si plusieurs arrêtes orientées sortent d'un même sommet, alors le suivant à ce sommet, est le sommet sur lequel arrive la dernière arrête orientée parmi ces arrêtes.
