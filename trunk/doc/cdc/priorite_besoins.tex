\subsection{Objectifs secondaires}

Les objectifs secondaires du projet sont les besoins du client qui ne sont
pas prioritaires et dont la réalisation sort du cadre des engagements de
l'équipe vis à vis du client.

Les tâches suivantes seront donc réalisées si le projet a pu être avancé plus 
rapidement que nous l'avions initialement prévu.


\subsubsection{Statistiques temps réel}

Le client désirerait pouvoir consulter les statistiques de chaque agent, de chaque
noeud, ainsi que celle de l'exécution de l'algorithme, et cela en temps réel. 
Ces statistiques pourront être intégrées dans une fenêtre séparée du
programme.

\subsubsection{Choix d'action lors de la suppression d'une arrête}

Le client voudrait pouvoir choisir dans la configuration d'une simulation
l'action à effectuer lors de la suppression d'une arrête sur laquelle circule
un agent parmis les cas suivant :\\

- 1) tuer l'agent (cas par défaut)
- 2) retourner l'agent vers son sommet d'origine


\subsubsection{Choix d'action de l'arrivée d'un agent sur un sommet éteint}

Le client voudrait pouvoir choisir dans la configuration d'une simulation
l'action à effectuer lorsqu'un agent se déplace dans un sommet éteint
(suite à un crash ou à une extinction) parmis les cas suivant :\\

- 1) tuer l'agent (cas par défaut)
- 2) retourner l'agent vers son sommet d'origine


\subsubsection{Rédaction d'une documentation sur la conception d'un algorithme dans Visidia}

La rédaction d'un algorithme dans ViSiDiA n'est pas une tâche facile et est
très peu documentée. Le client voudrait qu'une documentation détaillant comme rédiger
un algorithme dans ViSiDiA soit réalisée.
