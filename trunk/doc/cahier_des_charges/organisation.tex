\subsection{Utilisation d'une plate-forme de développement}

Le projet sera l'occasion d'utiliser un portail évolué de
développement sur le modèle du célèbre portail
\href{http://sourceforge.net}{SourceForge}. Notre choix s'est porté
sur le portail \href{http://www.berlios.de}{\berlios} pour des
raisons de rapidité des serveurs. Notre projet \visidia dispose ainsi
d'une page d'accueil
\href{http://visidia.berlios.de}{\visidia.\berlios} \cite{visidiaPFA}.

Cette page d'accueil est un wiki \cite{wiki}. Cela permet une
évolution plus souple des informations présentées. Cette page est le
point de départ vers les différents services fournis par \berlios.

Tout d'abord les sources de notre extension de \visidia sont
maintenues grâce au gestionnaire de version Subversion
\cite{subversion}. Le client a la possibilité de suivre l'avancée du
projet en accédant aux sources. Deux interfaces web permettent l'accès
aux sources: \href{http://svn.berlios.de/viewcvs/visidia/}{viewcvs} et
\href{http://svn.berlios.de/wsvn/visidia/}{websvn}. Il est également possible de télécharger anonymement
les sources depuis le dépôt Subversion:
\verb{svn checkout svn://svn.berlios.de/visidia/trunk}
